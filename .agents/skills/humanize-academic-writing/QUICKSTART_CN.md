# 快速开始指南

[🇬🇧 English](QUICKSTART.md) | [🇨🇳 中文](QUICKSTART_CN.md)

5分钟快速上手 Humanize Academic Writing。

> **📝 说明**：本指南用中文讲解操作步骤，但工具是用于改进AI生成的**英文**学术文本，所有示例都是英文。

---

## 🚀 安装

### 前置要求

- **Python 3.7+** （运行脚本需要）
- **Cursor IDE** （使用AI技能需要）
- **无外部依赖** - 仅使用Python标准库

### 方法1：Cursor 用户（推荐）

1. **复制技能到Cursor目录：**

**Windows:**
```powershell
xcopy "humanize-academic-writing" "%USERPROFILE%\.cursor\skills\humanize-academic-writing\" /E /I
```

**macOS/Linux:**
```bash
cp -r humanize-academic-writing ~/.cursor/skills/
```

2. **重启Cursor**

3. **测试:** 打开任何文本文件，对AI说："帮我人性化这段学术写作"

### 方法2：仅使用脚本

无需安装！直接运行：

```bash
python scripts/ai_detector.py tests/sample_ai_text.txt --detailed
```

## 🧪 第一次测试：检测AI模式

尝试包含的示例文件：

```bash
python scripts/ai_detector.py tests/sample_ai_text.txt --detailed
```

你会看到类似这样的输出：

```
======================================================================
AI 写作模式检测报告
======================================================================

整体AI概率: 高 (0.85)
建议: 文本显示强烈的AI模式。建议大幅改写。

详细分析:
----------------------------------------------------------------------

1. 句子均匀性: 🔴 高度关注
   平均句长: 16.8词，标准差: 2.3
   → 问题: 句子长度过于均匀
   → 修复: 混合短句(5-10词)、中等(15-20词)和长句(25-35词)

2. 机械过渡词: 🔴 高度关注
   12句(48.0%)以机械过渡词开头
   → 发现的过渡词: moreover, furthermore, additionally, in addition
   → 修复: 用隐式连接或变化的过渡替换

3. 抽象语言: 🔴 高度关注
   发现18个抽象短语 (密度: 每100词8.12个)
   → 最频繁: various (6次), multiple factors (4次), different (5次)
   → 修复: 用具体概念、具名理论、具体例子替换
```

## 💬 在Cursor中使用

### 方法1：自然请求

1. 打开AI生成的文本
2. 选中一个段落
3. 对AI说："请为社会学论文人性化这段学术写作"
4. Cursor会自动检测并应用此技能

### 方法2：特定模式修复

针对特定问题：
- "从这段落中删除机械过渡词"
- "用具体概念替换抽象语言"
- "变化这部分的句子长度"
- "为这个主张添加学术限定语"

### 方法3：按章节处理

- "为政治学论文人性化这个引言"
- "让这个方法部分听起来更自然"
- "改进这个文献综述——听起来太AI了"

## 🔄 示例工作流

### 步骤1：分析

```bash
python scripts/ai_detector.py my_draft.txt --detailed > analysis_report.txt
```

### 步骤2：查看问题

打开 `analysis_report.txt` 查看需要修复的模式。

### 步骤3：在Cursor中改写

打开Cursor并粘贴有问题的段落：

**你:** "这是我论文中的一段。AI检测器发现了高度机械过渡词和抽象语言。能帮我人性化吗？

[粘贴段落]"

**Cursor:** 将应用技能并进行改写并说明理由。

### 步骤4：验证改进

```bash
python scripts/text_analyzer.py my_draft.txt my_revised.txt --compare
```

## 📝 常见使用场景

### 场景1：引言段落

**问题:** "我的引言有太多'Moreover, Furthermore, Additionally'"

**Cursor中的解决方案:**
```
"这个引言有太多机械过渡词。
请用自然流畅的方式为社会学论文改写：

[粘贴引言]"
```

### 场景2：抽象语言

**问题:** 太多"various aspects"和"multiple factors"

**Cursor中的解决方案:**
```
"这段话太抽象。请用具体概念替换模糊短语：

[粘贴段落]"
```

### 场景3：句子均匀

**问题:** "所有句子长度相同"

**Cursor中的解决方案:**
```
"变化这段话的句子节奏
(混合短句、中等和长句)：

[粘贴段落]"
```

## 💡 获得最佳效果的技巧

### 要做的 ✅
- 从AI检测器开始识别具体问题
- 提供上下文："这是社会学论文的社交媒体部分"
- 要求理由："解释什么让这听起来像AI"
- 对复杂部分逐段处理

### 避免的 ❌
- 不要一次粘贴整篇论文（分段处理）
- 不要要求"让它无法检测"（要求"让它更自然"）
- 不要期望一键修复（好的写作需要判断）

## 🔧 故障排除

### "Cursor中未检测到技能"
- 复制技能后重启Cursor
- 检查技能在正确目录 (`~/.cursor/skills/` 或 `.cursor/skills/`)
- 确保技能文件夹中存在 `SKILL.md`

### "脚本无法运行"
- 检查Python版本: `python --version` (需要3.7+)
- 在某些系统上使用 `python3` 而不是 `python`
- 确保在正确的目录中

### "输出看起来没有改进"
- 具体说明要修复的模式
- 提供学科上下文（社会学、政治学等）
- 查看 `docs/examples_CN.md` 了解预期质量
